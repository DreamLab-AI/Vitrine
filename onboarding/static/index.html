How to prepare the Google Drive folder
  1. Put the capture videos in a single Drive folder (one sub-folder per session is fine).
  2. Copy the folder URL from the browser address bar (https://drive.google.com/drive/folders/<id>) and paste it above.
  3. Vitrine reads — and, with write-back enabled, writes — the folder through a Google service account (created in Step 4). After creating it, share this folder with the service-account email (…@<project>.iam.gserviceaccount.com): grant Viewer for read-only, or Editor if finished outputs are written back to the same folder.

Step 4 — Secrets

The manifest stores only env-var references (env:NAME), never raw tokens. If you paste a token below it is sent once to this local server, written to a 0600 .secrets.env file, and never returned to the browser.

Leave blank if the env-var is already exported on the host.

How to get your Hugging Face token
  1. Sign in at huggingface.co (a free account is enough).
  2. Open Settings → Access Tokens and click Create new token.
  3. Choose type Read (or a fine-grained token with read access to gated repos), name it vitrine, and create it.
  4. Copy the value (it starts hf_…) into the field above, or export it on the host as HF_TOKEN.
  5. Accept the model licences below — gated downloads fail without acceptance, even with a valid token.
Model licences to accept (open each page, click “Agree and access”)
  • FLUX.2-dev — inpainting / recovery (non-commercial licence).
  • Hunyuan3D-2.1 — fallback hull generator.
  • SAM 3D Objects — optional hull fallback (non-commercial).
  • gemma-4 and TRELLIS.2 are ungated — no acceptance needed.

Non-commercial models are flagged by the pre-run idiot-check; a commercial run must swap them (CoMe→PGSR, FLUX.2→Qwen-Image-Edit).

How to get Google Drive credentials (service account)
  1. Open the Google Cloud Console; create or select a project and note its Project ID (paste it above).
  2. Enable the Google Drive API for that project.
  3. Go to APIs & Services → Credentials → Create credentials → Service account; name it and create.
  4. Open the service account → Keys → Add key → Create new key → JSON; download the key file.
  5. Put the JSON on the host and point GOOGLE_APPLICATION_CREDENTIALS at its path. The same file is referenced by secrets/rclone.conf as service_account_file for the rclone gdrive remote.
  6. In Drive, share the source folder with the service-account email (Step 2) — Editor if outputs are written back, otherwise Viewer.

The browser never sees the JSON: the manifest stores only env:GOOGLE_APPLICATION_CREDENTIALS.
